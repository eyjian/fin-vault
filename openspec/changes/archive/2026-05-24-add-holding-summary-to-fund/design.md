## Context

当前基金页面（`FundManage.vue`）的列表表格仅展示基础信息（代码、名称、类型、基金公司、经理、净值等），缺少持仓汇总数据。而其他资产类型页面（股票、理财、现金）均已展示盈亏、市值等数据。

用户需要在基金列表中直接查看持仓盈亏情况，无需跳转到其他页面。

**当前实现差异**：
- StockManage.vue: 已传 `include_holdings: true`，已展示 8 个持仓汇总列
- FundManage.vue: 未传 `include_holdings`，未展示任何持仓汇总列

## Goals / Non-Goals

**Goals:**
- 基金列表表格添加持仓汇总数据列（持有数量、平均成本、市值、盈亏等）
- 保持与股票页面一致的展示逻辑和格式化方式
- 复用已有的后端 `holding_summary` API 和前端 `AssetHoldingSummary` 类型

**Non-Goals:**
- 不修改后端代码（已完全支持）
- 不修改数据结构或 API 接口
- 不涉及新功能（仅前端展示增强）

## Decisions

### 决策 1：直接复用 StockManage.vue 的列定义和格式化函数

**选择**：复制 StockManage.vue 的持仓汇总列定义到 FundManage.vue

**理由**：
- 数据结构和字段完全一致（都使用 `holding_summary` 对象）
- 格式化逻辑相同（盈亏颜色、百分比格式等）
- 保持用户体验一致性

**替代方案**：
- 抽取公共组件 → 过度设计，当前只有 2 个页面需要，未来可重构

### 决策 2：列的顺序安排

**选择**：在"最新净值"列之后插入持仓汇总列

**理由**：
- 净值信息是基金的核心数据，应靠前展示
- 持仓汇总（盈亏、市值）是衍生数据，放在基础数据之后更合理
- 与股票页面保持一致的信息架构

## Risks / Trade-offs

**风险 1：没有持仓的基金显示空数据**
- **风险**：新录入的基金可能没有持仓记录，`holding_summary` 可能为 `null` 或字段为空
- **缓解**：使用可选链（`?.`）和默认值（`|| '-'`）处理，参考股票页面的实现

**风险 2：净值更新频率与持仓数据不同步**
- **风险**：净值（latest_nav）和持仓市值计算可能使用不同时间点的数据
- **缓解**：当前设计如此，属于已知限制；未来可通过 `latest_price_time` 字段增强

**Trade-off：表格水平滚动**
- 添加 8 列后，表格会出现水平滚动
- 可接受：股票页面也已这样做，用户习惯横向滚动查看更多信息
